# 贡献指南

感谢您对摇篮留言服务系统的关注！我们欢迎任何形式的贡献，包括但不限于：提交问题、改进文档、提交代码等。

## 1. 行为准则

### 1.1 基本原则
- 尊重他人
- 友善交流
- 专业讨论
- 共同进步

### 1.2 沟通规范
- 使用清晰、专业的语言
- 保持积极、建设性的态度
- 避免使用过激或不当言论
- 尊重不同的观点和建议

## 2. 提交问题

### 2.1 问题报告
- 使用清晰的问题标题
- 详细描述问题现象
- 提供复现步骤
- 附上相关截图或日志

### 2.2 问题模板
```markdown
## 问题描述
[详细描述问题]

## 复现步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]

## 期望行为
[描述期望的正确行为]

## 实际行为
[描述实际发生的行为]

## 环境信息
- 操作系统：[系统版本]
- 浏览器：[浏览器版本]
- 应用版本：[版本号]

## 其他信息
[其他相关信息]
```

## 3. 提交代码

### 3.1 开发流程
1. Fork 项目
2. 创建功能分支
3. 提交代码
4. 发起 Pull Request

### 3.2 分支管理
- **分支命名**
  - feature/功能名称
  - bugfix/问题描述
  - hotfix/紧急修复
  - release/版本号

- **提交规范**
  ```
  <type>(<scope>): <subject>

  <body>

  <footer>
  ```

### 3.3 代码审查
- 确保代码符合规范
- 添加必要的测试
- 更新相关文档
- 处理代码审查意见

## 4. 文档贡献

### 4.1 文档规范
- 使用 Markdown 格式
- 保持结构清晰
- 内容准确完整
- 及时更新维护

### 4.2 文档类型
- 技术文档
- 用户指南
- API文档
- 部署文档

## 5. 测试贡献

### 5.1 测试规范
- 编写单元测试
- 编写集成测试
- 编写端到端测试
- 保持测试覆盖率

### 5.2 测试用例
```python
def test_feature():
    # 准备测试数据
    test_data = create_test_data()
    
    # 执行测试
    result = process_feature(test_data)
    
    # 验证结果
    assert result.status == 'success'
    assert result.data is not None
```

## 6. 发布流程

### 6.1 版本发布
1. 更新版本号
2. 更新更新日志
3. 创建发布标签
4. 发布新版本

### 6.2 发布检查
- 功能测试通过
- 文档更新完整
- 依赖包更新
- 安全漏洞修复

## 7. 社区维护

### 7.1 社区管理
- 及时响应问题
- 定期代码审查
- 组织社区活动
- 维护社区氛围

### 7.2 贡献者支持
- 提供技术支持
- 解答问题
- 提供反馈
- 认可贡献

## 8. 许可证

### 8.1 代码许可证
- 使用 MIT 许可证
- 保留版权声明
- 包含许可证文件
- 更新许可证年份

### 8.2 文档许可证
- 使用 CC BY-SA 许可证
- 保留作者署名
- 允许商业使用
- 要求相同方式共享
